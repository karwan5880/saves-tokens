# Model and reasoning routing

## Method

Route bounded mechanical work—classification, extraction, formatting, or summarization—to a cheaper model/low reasoning effort when validated; reserve stronger models/high effort for ambiguity, design, deep debugging, and final verification.

## Caution

The objective is lowest cost per successful task, not lowest per-call price. Weak routing introduces retries and hidden defects. Record quality by task class before automating a route.
